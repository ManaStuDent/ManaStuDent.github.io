我们使用自己 iphone 的时候，很多时候都需要截屏给朋友分享自己的内容。但是千遍一律的截屏似乎又没有什么新意。今天我带来一篇教大家使用 Workflow 做带壳截屏的 Action。

Workflow 在 v1.4.4 里加入了一个新动作（Action），叫做 Overlay Image，它可以帮你把两张图片叠在一起，使得使用 Workflow 做带壳截屏更加的方便。

使用 Workflow 做带壳截屏。

首先，我们来看看 Overlay Image 这个动作 。它需要两个量，一个是 Input，一个是 Image，如下图。它会把 Input 作为背景图片，然后把 Image 的图片覆盖。如果关掉 Show Image Editor 这个选项，我们还能看到其他几个变量：

  • Position 是 Input 图片的的默认位置
  • Width 和 Height 是 Input 图片的大小，如果不填写，那么就表示 Input 图片的大小不变
  • Rotation 表示旋转角度
  • Opacity 表示透明度

获取截屏

首先我们使用 Get Latest Screenshots 来获取我们最后一次的截屏，用 Set varible 来保存截屏的变量，这个截屏作为 Overlay Image 中 Input 的图片。我们要为 Set varible 取一个变量名称，这里取名叫做 Screenshots。

获取第二张图片

我们这里先使用 Select Photos 来获取我们带壳截屏的背景图片，就是那个”壳”。Select Mulltiple 可以让我们选取多个图片，这里我们只需要1张图片作为背景，所以这个就不用开启。

制作带壳截图

这里我们就要使用我们今天的主角–Overlay Image。这里 Overlay Image 放在 Select Photos 的下面，这样 Overlay Image 就获取了 Input 图片变量。然后我们在 Overlay Image 中 Image 的选项后面填入我们的第一个变量– Screenshots。(Set varible的名称)

Show Image Editor 这个选项我们不用打开，如果使用 Show Image Editor 的话，那我们每次使用这个 Action 的时候都需要我们手动调整两张图片的位置，和我们使用 Workflow 的初衷有点违背。Width、Height、Rotation(Degress) 和 Opactiy 我们都不需要设定。

那我们该如何保证2张图片形成完美的一张带壳截屏呢？这里需要我们准备的”壳”(iPhone6s外壳.jpg)的完美大小。只要我们准备的壳的内部屏幕大小和我们手机截屏的大小完美匹配，就不需要我们在 Overlay Image 进行编辑了。这里我做好了一个iphone6/6s的外壳素材，需要的戳这里 iPhone6s外壳.jpg，提取密码为:qn84。

开始我们的 Action

最后我们在 Overlay Image 的下面加入一个 Quickly look 来预览我们的成果。

进阶

我们使用 Workflow 的目的就是减少我们机械化的操作流程，从而解放我们的双手。但是看我们现在的这个动作，不免有点麻烦，我们使用这个 Action 后，还需要手动的选择一张照片，最后在 Quickly look 后还需要手动的保存照片。感觉这样还没有那些带壳截屏的软件来的简单。那么我们就动手改进我们的 Action 吧。

使用 Get File 来替换 Select Photos

当我们使用 Select Photos 的时候，系统会跳转到照片库中，让我们手动选择一张照片用来处理。但是我们使用 Workflow 的时候，有一个更加方便的动作来实现– Get File。

Get File 中有这几个变量：

  • Show Document Picker
  • iCloud File
  • Error If Not Found

这里为了让 Workflow 自动选择图片，所以在 iCloud File 中填入 iPhone6s外壳.jpg(我们事先将这张图片放入iCloud Driver/Workflow 文件中)，这样 Workflow 就可以自动的找到 iCloud Driver/Workflow/iPhone6s外壳.jpg 这张图片，从而不用让我们自己选择图片库中的图片，让整个动作更加流畅便利。

无法找到该文件

这里不知道是 Workflow 的问题，还是我对它的理解不够深刻。我在 iCloud Driver 中手动创建 Workflow 文件夹，并且把 iPhone6s外壳.jpg 放入其中后 iCloud File 就是找不到 iPhone6s外壳.jpg 这个文件，从而无法实现我的需求。

解决方法–让 Workflow 自己在 iCloud Drive 中创建 /Workflow 这个文件夹

这里就要使用到 Workflow 中的一个动作– Save File。当我们使用 Save File 这个动作保存一个文件，它默认的保存位置就是 iCloud Driver/Workflow/ 中，如果我们原本没有这个文件夹，那么 Workflow 就会为我们创建这个文件夹。当 Workflow 创建完这个文件夹后我们就可以把 iPhone6s外壳.jpg 这个文件放入其中，从而让 Get Flie 动作从 iCloud Driver/Workflow/ 中找到 iPhone6s外壳.jpg。

仔细对比 Workflow 创建的 /Workflow 文件夹和我们自己创建的 /Workflow 文件夹，我们不难发现二者图标是不一样的，而且这两个文件夹可以同时存在，或许这两个文件夹还是不一样的。

自动保存完成的带壳截屏

这里为了让 Workflow 自动保存完成的带壳截屏，我们可以使用上面用到的 Save File 这个动作，就可以让完成的带壳截屏保存到 iCloud Driver/Workflow/ 中。

或者使用 Save to Photo Album 动作保存到相册中。

删除原来的截屏

当我们使用 Workflow 生成带壳截图后，我们通常就用不到原来的截屏了，这个时候我们使用 Get varible(Screenshots)来获取截屏变量，然后使用 Delect Photo 动作来删除相册中的截屏。

总结

使用 Workflow 可不仅仅能够完成带壳截屏这样的动作，还有很多很多的动作可以为我们使用，帮助我们提高生产力，解放我们的双手，等你去发现！
