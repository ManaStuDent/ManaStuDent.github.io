Linux之旅一：安装，上网，ftp

2016-08-06

文章目录

  1. 安装
  2. 上网
  3. ftp

最近迷上了Linux，于是乎就安装了一个以 Oracle VM VirtualBox + VboxHeadlessTray + putty 的一个命令行Linux界面。

安装

安装的Linux发行版本是CentOS6.8的版本，手头上正好有，就拿来用了。安装过程还是比较顺利的，没有遇到什么问题。毕竟安装虚拟机还是很简单的，没有磁盘分配的一些问题，但是估计到时候在真机上面安装可能会有些问题。有问题的时候就去解决就好了嘛。

上网

如何让虚拟机的系统上网，是我遇到的第一个问题。VboxHeadlessTray可以打开CentOS的命令行界面，但是使用putty却无法通过SSH连接虚拟机。很自然的就想到是虚拟机没有网络的原因。

Oracle VM VirtualBox环境下
首先设置虚拟机的网络连接方式为桥连，并且在高级下面将连接网线打勾

显示是没有ip地址的。

修改ifcfg-eth0 设置 ONBOOT=yes 和 BOOTPROTO=dhcp

加载eth0,然后重启网络

（只输入config的话就没有etho，只有lo）：说明eth0网卡没有启动，启动方法是：ifup eth0
再次查看就会有eth0的Ip了，要是还或得不到IP地址，要检查VM的DHCP服务是否开启，若开启还是没有IP，要查看配置文件vi /etc/sysconfig/network-scripts/ifcfg-eth0里面的选项是不是dhcp获得。

ftp

因为是命令行界面，所以下载文件很不方便，于是我就想配置一个ftp来连接虚拟机。同样的和上网一样，遇到了很多的问题。

1.安装vsftpd软件

2.查看vsftpd 启动状态(这里还不是很清楚为什么开启了服务 chkconfig中还是off)

3.允许匿名用户，我们去掉配置文件（/etc/vsftpd/vsftpd.conf） 里面

前面的#号

4.允许root用户上传
修改/etc/vsftpd/user_list文件将root去掉
修改/etc/vsftpd/ftpusers文件将root去掉

5.将用户(一般指虚拟用户)限制在自家目录
修改配置文件（/etc/vsftpd/vsftpd.conf）去掉前面的#号
chroot_local_user=yes
这样用户就只能访问自己家的目录了

6.添加开放21号端口

7.修改上传文件夹权限

8.设置 SELinux

执行结果:ftp_home_dir–>off,setsebool allow_ftpd_full_access–>off
进行修改:# setsebool ftp_home_dir 1
进行修改:# setsebool allow_ftpd_full_access 1
修改后重新查看结果:ftp_home_dir–>on,allow_ftpd_full_access–>on

或者setenforce 0可以让SELinux变成宽容模式。

选项与参数：
0 ：转成 permissive 宽容模式；
1 ：转成 Enforcing 强制模式
