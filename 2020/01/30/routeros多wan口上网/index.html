routeros多wan口上网

2020-01-30

随着宽带变成每个人的必需品之后，家庭宽带也变得越来越平常。从原来的 2M 的小水管，到现在动辄 200M 甚至 1000M 的家庭带宽，如何利用好每一份资源变得越来越重要。前些日子家里办理了每月 10 块，合约期 2 年的移动 100M 宽带。虽说家里已经有了一个电信 50M 的宽带，奈何它只有 50M，有的时候家里人在玩游戏，但是我又要下载电影啥的，电信的宽带就隐隐约约的显得不是那么够用了。

办理宽带后就发现一个很头疼的事情，电信和移动两条光纤，两个猫，就是完全不相干的两个网络环境。我家里的网络设备原本的拓扑如下：

可以发现一个非常尴尬的问题，两个宽带将家里的设备都隔离开了，并没有办法统一的整合在一起，这里只是简单的描述了，只举例了一个 Nas 这个最具有代表性的东西。我的 Nas 是群晖的 DS218+，它是一个家用入门的 Nas，两盘位，然后只有一个千兆的以太网接口。所以它并不能同时使用电信或者的移动的网络。

下面是改造后的网络拓扑图：

这里我使用 Mikrotik RB4011iGS+ 同时接入电信和移动的宽带，然后使用不同 IP 地址使用不同的宽带的策略进行使用，电信作为默认的宽带使用，移动的需要手动指定 IP 进行使用。这里我将 IP 地址为 192.168.50.201-192.168.50.210 的设备使用移动宽带接入互联网，其他的默认使用电信宽带接入互联网。

比如我划定 192.168.50.1-192.168.50.200 作为 DHCP 服务的地址池，然后 ASUS AC68U 不再作为路由器，而充当无线 AP 使用。这样设备如果是自动获取 IP 地址，那么就默认的使用电信的网络。而设备想使用移动宽带就只需要手动指定设备的 IP 地址为 201-200 之间的任意一个即可使用移动宽带进行上网。

这里最重要的就是 routeros 的设置。这里简单记录下设置过程，以备后用。
1.通过电信 pppop 拨号的方式将电信的光纤宽带连入互联网，这里电信的猫为桥接，路由器拨号上网。这里就不深入记录了，网上应该有很多的教程。

2.划分一个移动的 wan 口，用于拨号。
我的设备一共 10 个网口和一个 SPF+ 口，默认 1 号口是 wan 口，其他的 10 个口做了一个 Bridge，默认都是 lan 口。
将一个网口，比如 6 号口，从 Bridge 中删除，然后修改名称为 wan2。

3.新增移动 pppoe
Interfaces -> add PPPoE Client

General -> Interfaces 选择 wan2
Dial Out -> 填写 User 和 Password
不要勾选 Add Default Route

4.标记 201-210 IP 地址的数据
IP -> Firewall -> Mangle -> add

General -> Chain 选择 prerouting Src. Address 填写 192.168.50.201-192.168.50.210
Action -> Action 选择 mark routing New Routing Marking 填写 移动routering

5.删除或者修改 Filter Rules 中的 fasttrack（注意点）
如果你用的是 Mikrotik 的物理设备，那么出厂就会有这么一条 fasttrack 的规则在 Firewall 中，默认是全部流量过滤。但是 fasttrack 和 Mangle 是冲突的，所以这里需要删除该条规则，或者修改该规则的 In. Interface 为电信的那个 pppoe-out1，也就是只过滤电信那个 wan 口的数据，而不过滤移动的。

6.添加 route
IP -> Routes -> add

General -> Gateway 选择 移动的那个 pppoe-out2 Routing Mark 选择 移动routering（第4步最后哪里填的这里就会出现）

注意点：
1.增加第二条宽带时不要勾选 Add Default Route
2.注意防火墙规则中的 fasttrack
