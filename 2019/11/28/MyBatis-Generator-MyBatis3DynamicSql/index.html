对 Java 和 Kotlin 都增加了一个使用 MyBatis Dynamic SQL 的 Runtime，并且将 MyBatis Dynamic SQL 作为默认的 runtime，真的是出场就是主角啊。使用 Java8 的语法，不再支持 iBatis2，增加代码清理功能等。

总之，就是比之前更加强大，最令人兴奋的是这一次将不再生成 .XML 文件，也就是说 .XML 的时代将慢慢的取消，取而代之的将会是一个颠覆之前 MSG 的风格。

MyBatis Dynamic SQL Usage Notes

MyBatis3DynamicSql 将为每张表生成 3 个类：
1.实体类，和数据库表一一对应，注意这里将不再单独生成 BLOBs
2.Support 类，该类用于辅助查询
3.mapper 类，该类提供了增删改查的通用方法
